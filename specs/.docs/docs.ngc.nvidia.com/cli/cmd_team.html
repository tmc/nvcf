team¶

Team Commands

usage: ngc team [--debug] [--format_type <fmt>] [-h]  ...

Named Arguments¶

--debug

Enable debug mode.

--format_type

Possible choices: ascii, csv, json

Specify the output format type. Supported formats are: ['ascii', 'csv', 'json']. Only commands that produce tabular data support csv format. Default: ascii

team¶

Possible choices: add-user, delete-invitation, info, list, list-users, remove-user, update-user

Sub-commands¶

list¶

List all accessible teams.

ngc team list [--column <column>] [--debug] [--format_type <fmt>] [-h]

Named Arguments¶

--debug

Enable debug mode.

--format_type

Possible choices: ascii, csv, json

Specify the output format type. Supported formats are: ['ascii', 'csv', 'json']. Only commands that produce tabular data support csv format. Default: ascii

--column

Specify output column as column[=header], header is optional, default is id[=Id]. Valid columns are name[=Name], description[=Description]. Use quotes with spaces. Multiple column arguments are allowed.

add-user¶

(For administrators only) Add existing user or invite new user to the current team.

ngc team add-user [--debug] [--format_type <fmt>] [--name <name>]
                  [--org-role <org_role>] --role <r> [-h]
                  <email|id>

Positional Arguments¶

<email|id>

User Email or ID

Named Arguments¶

--debug

Enable debug mode.

--format_type

Possible choices: ascii, csv, json

Specify the output format type. Supported formats are: ['ascii', 'csv', 'json']. Only commands that produce tabular data support csv format. Default: ascii

--name

User Display Name. Only required for new users.

--org-role

(For org administrators only) Specify the user role in the org, required to add a new user in the org. Updates user roles in the org if the user exists in the org. Options: [BASE_COMMAND_ADMIN, BASE_COMMAND_USER, BASE_COMMAND_VIEWER, FLEET_COMMAND_ADMIN, FLEET_COMMAND_OPERATOR, FLEET_COMMAND_VIEWER, FORGE_PROVIDER_ADMIN, FORGE_TENANT_ADMIN, NVIDIA_AI_ENTERPRISE_VIEWER, OMNIVERSE_ADMIN, OMNIVERSE_READ, OMNIVERSE_USER, REGISTRY_ADMIN, REGISTRY_READ, REGISTRY_USER, USER_ADMIN]. To specify more than one org role, use multiple --org-role arguments.

Required named arguments¶

--role

Specify the user role. Options: [BASE_COMMAND_ADMIN, BASE_COMMAND_USER, BASE_COMMAND_VIEWER, FLEET_COMMAND_ADMIN, FLEET_COMMAND_OPERATOR, FLEET_COMMAND_VIEWER, FORGE_PROVIDER_ADMIN, FORGE_TENANT_ADMIN, NVIDIA_AI_ENTERPRISE_VIEWER, OMNIVERSE_ADMIN, OMNIVERSE_READ, OMNIVERSE_USER, REGISTRY_ADMIN, REGISTRY_READ, REGISTRY_USER, USER_ADMIN]. To specify more than one role, use multiple --role arguments.

update-user¶

(For administrators only) Update a user's roles in the current team.

ngc team update-user [--add-role <add_role>] [--debug] [--format_type <fmt>]
                     [--remove-role <remove_role>] [--role <role>] [-h]
                     <id>

Positional Arguments¶

<id>

User ID

Named Arguments¶

--debug

Enable debug mode.

--format_type

Possible choices: ascii, csv, json

Specify the output format type. Supported formats are: ['ascii', 'csv', 'json']. Only commands that produce tabular data support csv format. Default: ascii

--role

Specify the user role. Options: [BASE_COMMAND_ADMIN, BASE_COMMAND_USER, BASE_COMMAND_VIEWER, FLEET_COMMAND_ADMIN, FLEET_COMMAND_OPERATOR, FLEET_COMMAND_VIEWER, FORGE_PROVIDER_ADMIN, FORGE_TENANT_ADMIN, NVIDIA_AI_ENTERPRISE_VIEWER, OMNIVERSE_ADMIN, OMNIVERSE_READ, OMNIVERSE_USER, REGISTRY_ADMIN, REGISTRY_READ, REGISTRY_USER, USER_ADMIN]. To specify more than one role, use multiple --role arguments.

--add-role

Specify the user role to assign. Options: [BASE_COMMAND_ADMIN, BASE_COMMAND_USER, BASE_COMMAND_VIEWER, FLEET_COMMAND_ADMIN, FLEET_COMMAND_OPERATOR, FLEET_COMMAND_VIEWER, FORGE_PROVIDER_ADMIN, FORGE_TENANT_ADMIN, NVIDIA_AI_ENTERPRISE_VIEWER, OMNIVERSE_ADMIN, OMNIVERSE_READ, OMNIVERSE_USER, REGISTRY_ADMIN, REGISTRY_READ, REGISTRY_USER, USER_ADMIN]. To specify more than one role, use multiple --add-role arguments.

--remove-role

Specify the user role to remove. Options: [BASE_COMMAND_ADMIN, BASE_COMMAND_USER, BASE_COMMAND_VIEWER, FLEET_COMMAND_ADMIN, FLEET_COMMAND_OPERATOR, FLEET_COMMAND_VIEWER, FORGE_PROVIDER_ADMIN, FORGE_TENANT_ADMIN, NVIDIA_AI_ENTERPRISE_VIEWER, OMNIVERSE_ADMIN, OMNIVERSE_READ, OMNIVERSE_USER, REGISTRY_ADMIN, REGISTRY_READ, REGISTRY_USER, USER_ADMIN]. To specify more than one role, use multiple --remove-role arguments.

list-users¶

(For administrators only) List all users in the current team.

ngc team list-users [--column <column>] [--debug] [--email <email>]
                    [--format_type <fmt>] [--invited] [--joined] [-h]

Named Arguments¶

--debug

Enable debug mode.

--format_type

Possible choices: ascii, csv, json

Specify the output format type. Supported formats are: ['ascii', 'csv', 'json']. Only commands that produce tabular data support csv format. Default: ascii

--joined

Only list users that have joined.

--invited

Only list invited users.

--column

Specify output column as column[=header], header is optional, default is id[=Id]. Valid columns are name[=Name], email[=Email], roles[=Roles], created[="Created Date"], type[="Invitation Type"], firstLoginDate[="First Login Date"], lastLoginDate[="Last Activity"], idpType[="Sign In Method"]. Use quotes with spaces. Multiple column arguments are allowed.

--email

Filter users by email.

Examples¶

How to list teams?¶

$ ngc team list

+-----+------------+----------------------+
| Id  | Name       | Description          |
+-----+------------+----------------------+
| 393 | robot-team | Team for Integration |
|     |            | tests                |
+-----+------------+----------------------+

How to list teams using column arguments?¶

$ ngc team list --column name

+-----+------------+
| Id  | Name       |
+-----+------------+
| 393 | robot-team |
+-----+------------+

How to get team details?¶

$ ngc team info cosmos

----------------------------------------------------
  Team Information
    Id: 161
    Name: cosmos
    Description: Cosmos team
    Deleted: False
----------------------------------------------------

How to add user to an team?¶

$ ngc team add-user foobar@nvidia.com --team cosmos --role BASE_COMMAND_USER

User: foobar@nvidia.com added to team: cosmos as {'BASE_COMMAND_USER'}.
